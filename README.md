# Socket IPC 项目

一个基于TCP套接字的进程间通信（IPC）项目，包含服务器端和客户端实现，支持多客户端并发连接。

## 项目特点

### 🚀 服务器端特性
- ✅ **多线程并发处理** - 支持同时处理多个客户端连接
- ✅ **心跳机制** - 自动检测连接状态，防止僵尸连接
- ✅ **日志系统** - 详细的运行日志记录，支持日志级别
- ✅ **线程安全** - 使用互斥锁保护共享资源
- ✅ **连接数限制** - 防止服务器过载
- ✅ **优雅关闭** - 支持信号处理，安全关闭服务器
- ✅ **超时控制** - 自动断开超时连接
- ✅ **错误处理** - 完善的错误处理和恢复机制

### 🎯 客户端特性
- ✅ **自动重连** - 网络异常时自动重连服务器
- ✅ **心跳功能** - 可开关的心跳机制保持连接活跃
- ✅ **命令解析** - 支持多种交互命令
- ✅ **超时控制** - 接收超时设置，防止阻塞
- ✅ **用户友好** - 清晰的交互提示和日志输出

## 编译和运行

### 环境要求
- GCC 编译器
- Linux 系统（支持 POSIX 线程）

### 编译
```bash
# 编译服务器和客户端
make

# 或者分别编译
make server
make client

# 清理编译文件
make clean
```

### 运行
```bash
# 1. 启动服务器
./server

# 2. 启动一个或多个客户端
./client [服务器IP] [端口]
# 示例:
./client 127.0.0.1 8080
./client 192.168.1.100 8080
```

## 使用说明

### 服务器端
服务器启动后监听指定端口（默认8080），支持以下功能：
- 自动记录客户端连接信息到日志文件
- 实时显示连接状态和活跃连接数
- 支持Ctrl+C优雅关闭

### 客户端
客户端支持以下交互命令：
- **普通消息** - 直接输入消息与服务器通信
- **`quit` 或 `exit`** - 退出客户端程序
- **`heartbeat on/off`** - 开启/关闭心跳功能
- **`ping`** - 发送PING测试连接状态
- **EOF (Ctrl+D)** - 安全退出

## 技术实现

### 核心功能
1. **TCP套接字通信** - 使用标准的TCP套接字API
2. **多线程处理** - 每个客户端连接使用独立线程
3. **select模型** - 客户端使用select进行多路复用
4. **心跳保活** - 定期发送PING/PONG保持连接
5. **线程同步** - 使用pthread互斥锁保护共享数据

### 错误处理
- 网络异常自动重连
- 超时连接自动清理
- 内存泄漏防护
- 信号安全处理
